**Recommendation**
Keep API-002 narrow and boundary-owned. The best home for the initial public `reason` vocabulary is [backend/handlers/envelope.go](/workspace/socialpredict/backend/handlers/envelope.go:8), not `backend/errors`, because the live `{ok, reason}` envelope already lives there and `backend/errors/httperror.go` is legacy `{error: ...}` code with no active handler usage.

Use uppercase snake case to match the repo’s existing contract, add one shared boundary helper for auth error classification, and normalize only handlers that already emit `ReasonResponse`-style JSON. Do not broaden this task into fixing all plaintext `http.Error(...)` routes. For OpenAPI, document the initial shared vocabulary in `ReasonResponse` descriptions/examples, not a strict global enum yet, because the backend still has plaintext middleware/errors and route-local reason synonyms.
