**Contract Compatibility Assessment**

Do not tighten `reason` to a strict enum: there are 27 live reason strings in code and `ReasonResponse` is referenced in 67 response slots, so a closed enum would either be incomplete or breaking.

High risk: the admin homepage mapper currently treats unmapped 403 auth errors as `INVALID_TOKEN`. I reproduced a failing focused test with `go test ./handlers/cms/homepage/http -run 'TestAdminUpdate_Success' -count=1`, which returned `403 {"ok":false,"reason":"INVALID_TOKEN"}`.
