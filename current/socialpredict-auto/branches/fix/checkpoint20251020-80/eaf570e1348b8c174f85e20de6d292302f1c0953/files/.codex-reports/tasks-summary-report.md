## Follow-up Findings

### Market Dust

- Current status: market dust still exists and is still active in the current branch.
- Branch comparison: relative to `origin/main`, the default dust behavior did not materially change; it was moved from the older handler-layer math package into the domain and service path.
- Current market-details behavior: aggregate `MarketDust` is still a placeholder `1` point per sell under the default `ConstantSellDustCalculator`, so total dust is effectively `number of sells * 1`.
- Sell-time behavior: the sell flow computes a dynamic per-sale dust amount as `creditsRequested - saleValue` and enforces `MaxDustPerSale`, but that exact dust is not persisted onto the stored bet row.
- Practical consequence: the market-details path cannot reconstruct exact historical dust from persisted bets alone, so the overview uses the placeholder aggregate dust model instead of the true realized per-sale dust.
- Volume semantics: market details expose `TotalVolume = base signed volume + MarketDust`; dust is added back as a scalar liquidity adjustment rather than stored as a neutral synthetic bet.
